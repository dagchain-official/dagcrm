Authentication & Access Control

System ki security ki neenv

🔐JWT Login

🎯 KyaEmail/password se login → JWT token milta hai jo har request ke saath jaata hai.
💡 KyunStateless auth — server pe session store nahi rakhna padta, isliye scale karna aasan. Token apne aap refresh hota hai.
🔧 KaamHar user secure login kare; token expire ho to auto-refresh, warna re-login.

🛡️3-Layer Access Control

🎯 KyaLayer 1 Role (kaunsa dashboard/menu) · Layer 2 Business (kaunse business ka data) · Layer 3 Module CRUD (View/Create/Edit/Delete).
💡 KyunHar banda sab kuch na dekhe. RM ko payroll nahi, HR ko sales nahi. Confidentiality + clean UX + security.
🔧 KaamRM A → sirf 3 businesses; Sales Manager → Revenue dekhe par edit na kare; HR → sirf HR modules.
Login → role check → dashboard + filtered sidebar (L1)
      → business check → sirf allowed businesses (L2)
      → module check → buttons hide/show per permission (L3)

⚙️Permission Matrix (Admin)

🎯 KyaAdmin page jahan se har role × module ka permission toggle hota hai, aur per-user business access set hota hai.
💡 KyunPermissions code me hardcode na hon — admin bina developer ke UI se control kare. Company badle to bas toggle.
🔧 KaamSales Manager ko Revenue "Edit" dena ho → toggle ON → turant apply.

🔑Profile, Change & Forgot Password

🎯 KyaApni profile edit, password change, aur bhool jaye to email link se reset.
💡 KyunSelf-service — har choti cheez ke liye admin ko bolna na pade. Password reset bina IT help ke.
🔧 KaamNaya number update, password bhool jaye to "Forgot password" → email link → reset.

Har role ko apna view

📊7 Alag Dashboards

🎯 KyaLogin karte hi role ke hisaab se dashboard: Admin (company), Sales Manager (sales+targets), Team Leader (team), RM (my workspace), HR, Finance, Support.
💡 KyunGeneric dashboard bekaar — RM ko apne leads chahiye, HR ko attendance, Finance ko profit. Sabko relevant view → productivity.
🔧 KaamRM login → "My Leads, My Revenue, Follow-ups"; HR login → "Present today, Pending leaves, Payroll".
Login → role → us role ka dashboard component
   Admin/BH → Company   Sales Mgr → Sales+Targets
   Team Lead → Team     RM → My Workspace
   HR → People          Finance → Money   Support → Tickets

Lead Management

Sales ki shuruaat

👤Leads (CRUD + Filters + Search)

🎯 KyaPotential customers ka record — naam, phone, source, status, owner, AI score.
💡 KyunLead = future paisa. Track na ho to leads thande ho jate hain, sales miss.
🔧 KaamNaye lead add, status/source se filter, naam/phone se search.

📥CSV Import

🎯 KyaCSV file se bulk leads upload — auto dedup, source auto-create, validation.
💡 KyunPurane data ya offline events ke leads ek-ek manually daalna slow. Dedup taake duplicate na aaye.
🔧 KaamExhibition ke 500 leads CSV me → ek click import → duplicates skip.

🔀Lead Distribution

🎯 KyaUnassigned leads RMs me auto-baant: Round Robin (barabar), Performance (top RM ko zyada), Manual (specific RM).
💡 Kyun100 leads aaye, manually baantna slow + unfair. Auto-distribute fair aur fast. RM ko turant notify.
🔧 KaamSubah ke naye leads → Distribute → har RM ko apne leads + notification.
Unassigned leads → Distribute → strategy
  Round Robin: fewest-load RM ko next lead
  Performance: high-conversion RM ko zyada
  Manual: ek RM ko sab
→ assign + 🔔 RM notify

🎯AI Lead Scoring & Auto Status

🎯 KyaHar lead ka 0-100 score (kitna "hot"). Aur engagement pe status apne aap aage badhe.
💡 KyunRM ko pata ho kis lead pe pehle focus kare (high score). Status manually badalna bhool jate hain → auto.
🔧 KaamScore 85 wala lead pehle call; call karte hi status new→contacted apne aap.
Call/WhatsApp/Email  →  status: new → contacted
Meeting / Proposal   →  status: → qualified
(converted/lost = terminal, override nahi)

Engage & Telephony (Twilio)

CRM se hi direct contact

📞Call / WhatsApp / Email / Proposal

🎯 KyaLead detail page se ek click me Call (Twilio), WhatsApp, Email, ya Proposal bhejo. Har action auto activity me log + status update.
💡 KyunRM ko alag phone/WhatsApp na kholna pade — CRM se hi sab. Aur manually activity likhna bhool jate hain → auto-log.
🔧 KaamLead kholo → "Call" → Twilio connect kare → activity me "call" + status contacted → timeline me dikhe.
Lead detail → "Call"
  → Twilio: agent ko ring → lead se connect (live)
  → LeadActivity(call) create
  → signal: status new→contacted (auto)
  → timeline + toast
(Twilio creds na ho to: activity log hota hai, status update hota hai — bas live call nahi)

Support Desk

Customer issues

🎫Tickets + SLA + Comments

🎯 KyaCustomer issues = tickets (priority/status). Detail page me SLA tracker, comments thread, attachments.
💡 KyunIssues time pe solve hon. SLA target (urgent=4h) batata hai kaunsa late ho raha → "Breached" red → priority.
🔧 KaamUrgent ticket → SLA "2h left" → agent jaldi solve; comments me update; file attach.
Ticket → priority → SLA target (urgent 4h / high 24h / med 48h / low 72h)
  elapsed vs target → On-track / Breached
  comments thread + attachments

Sales, Revenue & Targets

Paisa tracking

💵Revenue (Auto Net) & Targets

🎯 KyaRevenue record (Gross − Commission = Net, auto). Targets with achieved vs progress bar.
💡 KyunPaisa calculation me galti na ho — system kare. Targets se pata chale kitna achieve hua.
🔧 KaamGross 10000, Commission 1500 → Net 8500 auto. Target $50K, achieved $22.5K → 45% bar.

HR, Attendance & Leaves

People management

⏰Real-time Attendance + Auto Activity

🎯 KyaLive clock se Check In/Out. Background me active/idle time + calls/notes/tickets auto-count.
💡 KyunManual attendance jhoothi ho sakti. Real check-in + auto activity → fair performance picture.
🔧 KaamEmployee Check In → din bhar activity auto-track → HR ko sach me kaun active dikhe.

🏖️Leave Workflow

🎯 KyaEmployee khud apply kare → manager/HR approve → dono ko notify.
💡 KyunProper chain — employee apply, jo team chalata wahi (manager) approve, HR sab dekhe. Koi request atki na rahe.
🔧 KaamRM "My Leaves" → apply → Team Leader ko notify → approve → RM ko "approved" notify.
Employee apply (My Leaves)
   → 🔔 direct manager ko notification
   → manager/HR Leaves page me dikhe → Approve/Reject
   → 🔔 employee ko result notify
(peer dusre ki leave na dekhe na approve)

Payroll & Incentive Engine

Salary automation

💰Payroll + Payslip PDF

🎯 KyaBasic + Incentive + Bonus − Deduction = Net Pay (auto). Branded Payslip PDF download.
💡 KyunSalary calculation manual me galti-prone. Payslip har employee ko chahiye — proof of pay.
🔧 KaamPayroll entry → Net Pay auto → "Payslip" button → professional PDF.

🧮Incentive Auto-Calculation

🎯 KyaRevenue se RM ka incentive auto-compute (rule: %/fixed), phir payroll me flow.
💡 KyunHar RM ka incentive manually nikalna slow + galti. System: revenue → RM → rule → amount → payroll, ek click.
🔧 KaamMahine ke end "Auto-Calculate" → sab RMs ke incentive ban jayein + payroll update.
Revenue → customer → lead → RM (jisne deal kiya)
   → us business ka incentive rule (e.g. 10%)
   → incentive amount → Payroll incentive field → Net Salary auto-update

🔧 KaamFinance dashboard → revenue, expenses, profit ek nazar me.

Platform (Meta/Google/…) → webhook POST (secret se secured)

  → Lead create + source tag → auto-assign RM → 🔔 notify
