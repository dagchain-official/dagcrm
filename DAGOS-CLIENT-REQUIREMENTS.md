# DAGOS — What We Need From You To Connect Your Businesses

**Purpose of this document:** To bring all your businesses into one DAGOS dashboard, we need to "plug in" to each business's existing system so the numbers (sales, revenue, customers, etc.) flow into DAGOS automatically — instead of anyone typing them by hand.

This document explains, in simple terms, what we need you to provide for each business. You don't need to be technical — just point us to the right system and the right person, and give us permission to connect.

---

## First, the simple idea

Think of DAGOS as a **central control room**. Each of your businesses (Forex, DAGChain, DAGGPT, etc.) already runs on its own software. We need a secure "pipe" from each of those systems into DAGOS so the data updates on its own.

To build that pipe, for **every** business we need three basic things:

1. **Which software does this business run on?** (the name of the tool/platform you use today)
2. **Permission to connect to it** — usually a username/password or an access key that their system can generate for us. (Please share these securely, not over normal email/WhatsApp — we'll provide a safe way.)
3. **A contact person** who manages that software and can answer our questions.

If a business has **no software yet** and runs on spreadsheets, that's fine too — tell us, and we'll set it up so the team enters data directly in DAGOS.

## Rules & settings we need you to decide (very important)

DAGOS is built so **you control the rules — not the developers.** But to set it up the first time, we need you to tell us your current rules for:

- **Targets:** How do you set a person's target? (e.g. "target = their cost to the company × 2")
- **Incentives:** Your reward rules. Examples we'll set up exactly as you define them:
  - "If someone hits 100%–200% of target → 10% incentive"
  - "10 meetings done → $200 bonus"
  - "100 lots traded → $2 per lot"
  - "100 new users → $500 bonus"
- **Cost to Company (CTC):** What costs do you count per employee? (salary, visa, desk, internet, office, etc.)
- **Performance scoring:** How much weight goes to Revenue vs. Growth vs. Activity? (e.g. 60% / 25% / 15%)

You can change all of these yourself later inside DAGOS — we just need the starting values.
