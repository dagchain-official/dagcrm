1System kaise bana hai

DAG CRM 3 services + ek database pe chalta hai. Sab alag, par ek saath kaam karte hain.

Frontend

React + Vite + Tailwind

Poora UI — dashboards, forms, boards. http://localhost:5173. Data backend se API ke through aata hai, har 1–15 sec live refresh.

Main API

Django + DRF

Saara business logic, permissions, engines. http://localhost:8000/api/. JWT auth (login token).

AI Service

FastAPI

Lead scoring, dashboard insights, "AI over your database" assistant. http://localhost:8100.

Database

PostgreSQL / SQLite

Saara data. Local dev mein SQLite (db.sqlite3), production mein Postgres.

Core principle: "Nothing hardcoded." Admin har cheez — businesses, products, hierarchy, targets, KPIs, incentive rules, formulas — bina developer ke configure kar sakta hai.

Step-by-step kya hota hai

#StepFunctionality
1Lead captureLead 3 tareeke se aata hai: manual entry, webhook (Meta/Google ads), ya FXArtha API sync. Har lead ko ek RM auto-assign hota hai (load-balanced) + notify.
2Lead ActivityRM call/WhatsApp/email/meeting log karta hai, follow-up date set karta hai. AI lead ko 0–100 score deta hai. Status auto badalta hai (new → contacted → qualified).

Attribution: Revenue → Customer → originating Lead → assigned RM. Isi chain se har RM ka revenue pata chalta hai, aur manager tak roll up hota hai.

6Money engines — CTC → Target → P&L

Har employee ki cost, uska target, aur profit — sab derived, admin-configurable.

CTC Engine

# Cost To Company
CTC = salary
  + Σ cost categories
# (visa, desk, internet…)

Admin cost categories add karta hai. Har employee ki monthly CTC nikalti hai.

Target Engine

Target = CTC × Multiplier

# e.g. $2000 × 2 = $4000

Multiplier admin set karta hai (global / per-level / per-employee). Manager ka target = team ka sum.

P&L Engine

Profit = Revenue − Cost

# har level ka apna P&L
# RM → TL → Manager → Head

Revenue (net) minus CTC. Poore org tree mein roll up — har level ka profit + margin%.

Target BoardCTC · multiplier · target · achieved · progress% — hierarchy rollup
P&L StatementRevenue − Cost = Profit, har level pe

7KPI & Performance

Product-specific numbers track karo, aur employees ko score karo.

KPI Engine (configurable)

Admin har business ke apne KPIs define karta hai — bina code. Do type:

SourceMatlabExample
ManualValue khud enter karni hoti hai (KPI Entries)Lots Traded, Nodes Sold, New Deposits
DerivedCRM data se auto — koi entry nahiMeetings/Calls (Lead Activity se), Leads Converted

Aggregation-aware rollup: sum/count jud jaate hain, average count-weighted, latest = snapshot. Manager ke row pe team ka rolled-up number.

KPI DefinitionsAdmin metric banata hai (business, unit, aggregation, category)
KPI EntriesManual values record
KPI BoardSab KPIs, per business, hierarchy rollup

Performance — 3 Scorecards

Har employee 3 independent scorecards pe measure hota hai, phir admin-weight se ek overall score (0–100, peer-normalised — top performer = 100):

Revenue

Revenue generated + target attainment. Default weight 60%

Growth

New users, deposits, AUM, retention (growth-category KPIs). 25%

Activity

Meetings, calls, demos (activity-category KPIs). 15%

Overall = Revenue×0.60 + Growth×0.25 + Activity×0.15 # weights admin-configurable

Model 1 — Target Achievement Slabs

Attainment% (achieved ÷ target) jis slab mein giro, utna % incentive:

AttainmentIncentiveExample (revenue $10,000)
0% – 100%0%$0
100% – 200%10%$1,000
200% – 400%20%$2,000
400%+30%$3,000

Model 2 — Activity Incentives (per-KPI reward)

Kisi bhi KPI pe per-unit reward. Ye Part 6 KPIs ko directly use karta hai:

KPIRateExample
Lots Traded$2 / lot200 lots → $400
Meetings$20 / meeting10 meetings → $200
New Users$5 / user100 users → $500
Total incentive = slab amount + Σ activity rewards
Run payout kya karta hai: har employee ka total nikaal ke Incentive table mein likhta hai (source-tagged: slab/formula) + us mahine ki Payroll mein incentive field update → final salary auto recompute. Preview pehle, run baad mein.

9Formula Builder — no-code rules (spec: MANDATORY)

Admin apne khud ke rules banata hai — bina developer. Ye slab+activity ka general version hai + multi-condition (AND/OR).

Rule = conditions (named variables pe) → payout. Variables sab engines se aate hain:

revenue · cost · target · attainment · profit · kpi:<any-metric>

# Examples admin structured builder se banata hai:
IF revenue > cost × 2     → 10% of revenue
IF kpi:new-users > 50  → $500 flat
IF kpi:lots-traded > 100 → $2 per lot
IF kpi:meetings > 10    → $200 flat
Safe by design: koi eval() nahi (RCE risk). Rules structured hote hain — variable + operator + value. UI pe human-readable preview dikhta hai.

10AUM & Business Contribution

New AUM (Assets Under Management)

Client ka paisa jo manage ho raha hai. Track hota hai:

Net New AUM = New Deposits − Withdrawals
Closing AUM = Existing + Net New

Per business, org tree mein rollup (RM → Head). FX Artha ke liye FXArtha sync auto bharता hai.

Business Contribution

Client se kitna net faayda — deposit, trading loss, brokerage, insurance, staking.

Net = Σ (component × weight)
# brokerage+1, loss−1, deposit 0…

Weights admin-configurable — CRM formula hardcode nahi karta.

12Dashboards & Business Switcher

Har role apna dashboard dekhta hai. Upar ek dropdown se business-wise view bhi.

  • Role dashboards: Admin, Sales Manager, Team Leader, RM, HR, Finance, Support — har ek ke apne charts & cards.
  • Business switcher (dropdown): "All Businesses" = normal CRM view. Koi business select karo → us business ka dashboard — revenue trend, uske KPI cards, AUM, top RMs. Cards dynamically us business ki KPIs se — hardcoded nahi.
  • AI Insights: dashboard pe AI automatically 2–3 insights deta hai (data ke basis pe).
